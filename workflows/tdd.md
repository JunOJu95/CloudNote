# TDD (Test-Driven Development) Prompt 가이드라인

효율적이고 견고한 TDD 사이클을 유지하기 위해 AI 어시스턴트(또는 개발 팀)가 요구사항을 구현할 때 반드시 따라야 하는 TDD 지침 및 프롬프트 양식입니다. 기능을 개발할 때마다 아래의 규칙을 준수하여 진행합니다.

## 🌟 핵심 원칙: Red - Green - Refactor 페이즈 엄수

기능 구현은 생략 없이 반드시 다음 세 가지 페이즈를 거쳐야 합니다.

1. **Red (테스트 실패)**: 구현할 기능의 요구사항을 검증하는 실패하는 테스트 코드를 가장 먼저 작성합니다.
2. **Green (테스트 통과)**: 방금 작성한 테스트를 통과시킬 수 있는 최소한의 실제 코드를 작성합니다.
3. **Refactor (리팩토링)**: 테스트를 통과한 상태(안전망)에서 코드의 구조, 가독성, 성능을 개선합니다.

---

## 📋 TDD 실행 프롬프트 양식

새로운 기능을 구현할 때는 아래 프롬프트를 활용하여 작업을 지시합니다.

```markdown
다음 요구사항을 TDD(Test-Driven Development) 방식에 맞춰 구현해 주세요.

[요구사항 정의]
- (구현해야 할 기능이나 요구사항을 여기에 명시합니다.)

[TDD 구현 규칙]
1. **Red Phase**: 실제 코드를 작성하기 전에, 실패하는 테스트 코드를 먼저 작성해 주세요. 
2. **Green Phase**: 작성된 테스트가 통과하는 최소한의 실제 코드를 구현해 주세요.
3. **Refactor Phase**: 테스트가 통과하는 것을 확인한 후, 코드 품질 향상을 위해 리팩토링을 진행해 주세요.

[테스트 커버리지 및 종류]
- Unit Test(단위 테스트), Integration Test(통합 테스트), E2E Test(End-to-End 테스트)를 요구사항 문맥에 맞게 모두 포함하여 작성해 주세요.
- 테스트 커버리지는 항상 **80% 이상**을 유지하도록 테스트 시나리오를 꼼꼼하게 구성해 주세요.

[검증 및 사후 조치]
- 모든 구현(Red-Green-Refactor)이 완료된 후, 전체 테스트 스위트를 다시 실행하여 **모든 테스트가 통과하는지 반드시 확인**해 주세요.
- 만약 통과하지 못한 테스트가 발생할 경우, 실패 원인을 자세히 분석하여 설명하고 코드를 수정하여 최종적으로 100% 통과하도록 조치해 주세요.
```

## 🛠 각 테스트 단계별 권장 사항 (Next.js & Supabase 기준)

* **Unit Test**: 개별 유틸리티 함수(예: 날짜 포매팅, 금액 포매팅), React Hook 유닛 등을 검증합니다. (Vitest 또는 Jest 사용)
* **Integration Test**: 컴포넌트 간 상호작용, 커스텀 훅과 UI의 연결, Supabase DB 쿼리(로컬 모킹 사용) 등을 검증합니다. (React Testing Library 활용)
* **E2E Test**: 사용자 관점에서의 전체 흐름(예: 로그인 -> 상품 목록 -> 장바구니 추가 -> 결제 완료)을 브라우저 환경에서 검증합니다. (Playwright 또는 Cypress 사용)
